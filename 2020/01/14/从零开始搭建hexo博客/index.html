从零开始搭建hexo博客

此教程将介绍如何从一台未安装任何环境的windows电脑从零开始搭建hexo博客。

准备

下载 & 安装

事先需要下载以下安装包：

  1. Node.js
  2. Git

下载最新版本即可，Git下载页面在github上，选择最新版本的.exe文件即可。

下载好两个文件后可以直接安装，不用勾选任何其他的可选项，直接安照默认一路next。

安装完成后可以在键盘按下win + R输入cmd，回车，分别执行命令：node、npm、git。
查看是否安装正确。若出现以下回显则安装正确。


如果提示找不到命令可能是安装错误或者在安装之前就打开了cmd窗口，如果关闭cmd再试一次还是没有反应的话可能就是安装错误，在控制面板卸载后重新安装一遍。

下载hexo

可以使用npm命令就可以下载hexo了，在命令行中使用命令：npm install -g hexo。
其中-g参数是全局安装的意思，也就是自动配好环境变量在所有文件夹下都可以使用hexo命令。


安装hexo可能要花一点时间，在这段时间可以先找个位置创建一个文件夹，作为存放自己博客的文件夹。可以在命令行使用mkdir blog或者直接创建一个文件夹。

搭建hexo博客

本地搭建

建好一个文件夹后通过命令行进入文件夹：cd blog，接着执行hexo init命令即可初始化一个blog文件夹。注意执行hexo init命令时一定要是一个空文件夹，否则会报错。在初始化时也可能会花一点时间，耐心等待一下。

完成后可能会报几个WARN，没有关系，我们可以执行hexo s预览一下，其中s时server的意思，和hexo server的效果一样。

这时候博客已经运行在本地了，在浏览器中访问 http://localhost:4000 或者 http://127.0.0.1:4000 即可访问。

此时一个hexo博客就搭建完成了。

创建文章

创建一篇文章也很简单，在blog文件夹下使用命令：hexo new 你的文章名字。hexo就会自动帮你生成一篇默认的markdown文章。并且告诉了文章的位置：就在hexo文件夹->source文件夹->_post文件夹->你的文章名字.md。

实际上，要新建一篇文章也就是新建一个markdown文件，然后以markdown格式写好文章后，hexo会把它自动渲染成一个静态页面。此时再执行hexo server，访问 http://localhost:4000 ，会发现博客上多出了一篇test文章。

部署至github及coding

此时我们的博客已经可以在本地正常浏览了，那么如何把它发布到网上呢？github和coding为我们提供了静态网站部署的方法。

github

首先我们要在Github上注册一个账号，账号的名称以后会作为博客的域名，所以可以取一个好一点的名字，然后创建一个仓库用来存放博客的文件。注意这里仓库的名字一定要是你的Github名字.github.io。

接下来我们要做的就是将自己的电脑与Github上的仓库相连了，这时候我们需要在blog文件夹中用Git bash打开文件夹。

打开后需要配置一下用户的信息，在Git Bash中执行下面的两条命令：

  1. git config --global user.name '你的github名字'
  2. git config --global user.email '你的github邮箱'

接下来生成ssh key就可以在本地部署blog文件到github仓库了。

执行这条命令：ssh-keygen -t rsa -C 你的Github邮箱

在生成ssh key时会要求输入一些东西，全部默认也就是按下回车就可以了。注意它会告诉你生成的地址，在图中的地址也就是：/c/User/ufoqu/.ssh文件夹。

生成好以后我们应该就可以在这个文件夹下找到两个文件：id_rsa和id_rsa.pub。
其中的.pub文件就是publish的意思，也就是公钥。

我们在Github的仓库(repositories)中可以进行设置把刚才生成的ssh key添加进去，选择Deploy keys，选择添加一个key（Add deploy keys），然后把刚才的id_rsa.pub中的内容复制进去，这个文件可以通过记事本打开。

添加完以后就剩最后配置一下config文件了，在hexo博客文件夹下可以找到这样一个文件：_config.yml，可以用记事本打开。

这样所有的配置都完成了。这样我们就能把文章发布出去了。

coding

实际上如果把博客部署在github上访问速度是很慢的，因为毕竟是在国外，那么我们可以在国内找一个平台，比如Coding。其实在Coding中的配置和在github中也差不多，首先需要创建一个账号，团队版和个人版都可以，这里就用团队版做示范吧。

首先创建一个项目用来存放我们的代码，选择第一个项目模板，这里的项目名称不同于Github，可以任取。项目标识按照它默认的也就可以。注意要勾选公开源代码，这样别人才能访问你的博客，其他的默认就行了。

创建好项目以后在项目设置->项目与成员->功能开关中将构建与部署勾选。

这样在项目界面可以选择构建与部署->静态网站，在Coding中使用静态网站功能是需要实名认证的，认证完成后，创建一个静态网站，网站名称可以任取，其余都默认即可。

在完成之后我们需要和GitHub一样设置一下ssh公钥。可以直接使用刚刚生成的id_rsa.pub。直接把它复制到代码仓库->仓库设置->部署公钥->新建部署公钥。其中的名称可以任取。

最后我们再来修改一下_config.yml文件，将原来的github地址放到另外一行，同时coding也添加一行，这样我们在发布文章的时候就会同时在github和coding上同时发布了。

修改完成之后就全部ok了。

发布文章

现在我们可以发布文章了，首先新建一篇文章：hexo new 测试文章，然后可以往测试文章.md中添加内容。最后进行发布，发布之前我们还需要按照一个hexo插件，执行这条命令：npm install hexo-deployer-git --save。
